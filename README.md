## 🎯 产品定位

**智慧营销红包发放系统** 是一款专为企业打造的高性能微信红包发放管理平台，基于微信支付官方「商家转账到零钱」能力深度定制，完美适配课程推广返现、活动营销奖励、渠道佣金发放等多元化业务场景。

### ✨ 核心亮点

- 🚀 **秒级到账** - 用户确认收款后资金实时到账微信零钱
- 🔒 **金融级安全** - 多重风控机制，资金流转全程加密
- 📊 **合规审计** - 完整的业务链路追溯，满足审计要求
- 🎨 **品牌定制** - 多种精美模板，支持企业品牌视觉定制
- 👥 **权限分级** - 管理员与员工分级管控，数据隔离安全可控

## 🚀 快速开始

### 提前准备

- 已经ICP备案的域名。
- 已经花了300认证的微信公众号-服务号，类型必须是服务号。
- 已经开通了微信支付+商家转账功能。

### 环境要求

| 组件 | 最低版本 | 说明 |
|------|---------|------|
| PHP | 7.4+ | 需开启 mysqli、gd、curl 扩展 |
| MySQL | 5.7+ | 建议 InnoDB 引擎 |
| Web服务器 | Nginx/Apache | 需支持 HTTPS |
| 微信商户 | - | 需开通「商家转账到零钱」权限 |

### 一键部署

```bash
# 1. 克隆代码到服务器
cd /www/wwwroot
git clone https://github.com/jinghong-me/PromoPay.git

# 2. 导入数据库结构
cd redpacket-system
mysql -u root -p < database.sql

# 3. 配置目录权限
chmod -R 755 uploads/ logs/ cert/

# 4. 修改配置文件
vim config.php  # 修改数据库和站点配置
```

### 核心配置

```php
// config.php - 关键配置项

// 数据库配置
define('DB_HOST', 'localhost');
define('DB_NAME', 'redpacket_system');
define('DB_USER', 'root');
define('DB_PASS', 'your_secure_password');

// 站点配置
define('SITE_URL', 'https://your-domain.com');

// 微信支付配置（后台可视化配置，无需硬编码）
```

## 🛠️ 技术栈

### 后端架构
- **核心语言**: PHP 7.4+ (原生架构，零框架依赖)
- **数据库**: MySQL 5.7+ (PDO + 预处理语句)
- **支付网关**: 微信支付 API v3 (RSA-SHA256 签名)
- **缓存机制**: 文件缓存 + 数据库事务锁

### 前端技术
- **页面构建**: HTML5 + CSS3 (Flex/Grid 布局)
- **交互逻辑**: 原生 JavaScript (ES6+)
- **视觉设计**: CSS 变量 + 渐变动画
- **响应式**: Mobile-First 适配策略

### 安全机制
- **传输安全**: HTTPS 全站加密
- **数据安全**: SQL 预处理防注入、XSS 过滤、CSRF Token
- **支付安全**: 双向证书校验、签名验证

## 📖 使用指南

### 创建并发放红包

```
1. 登录后台 → 点击「创建红包」（或在后台首页快捷创建）
2. 选择红包类型（一次性/批量/多次领取）
3. 填写发放参数（金额、数量、有效期等）
4. 选择视觉模板 → 点击「立即创建」
5. 下载二维码 → 投放至目标渠道
6. 用户微信扫码 → 点击领取 → 资金秒到零钱
```

### 员工管理流程

```
管理员
  ├── 创建员工账号
  ├── 分配发放权限
  └── 查看发放统计

员工
  ├── 独立后台登录
  ├── 创建个人红包
  └── 查看个人业绩
```

```bash
# 日志轮转（建议每日执行）
```

## ❓ 常见问题

**Q: 用户反馈领取后未到账？**
> 请检查：1) 微信支付配置是否正确 2) 商户号是否开通转账权限 3) 查看系统日志定位具体错误

**Q: 如何防止恶意刷红包？**
> 系统已内置风控：单用户领取次数限制、IP 频率限制、设备指纹识别。可在后台调整阈值。

**Q: 是否支持企业微信？**
> 当前版本基于微信公众号/小程序 OpenID，企业微信适配版本正在开发中。
